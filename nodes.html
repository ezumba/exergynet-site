Step 02 — Identity

On first launch, enter your EVM payout address. This is your permanent USDC yield destination on Base Mainnet. Any standard EVM wallet is accepted.

Format: 0x...

Protect your private key locally. ExergyNet never holds or requests your key — only your public payout address is required.

Step 03 — Ignite

The Desktop Prover connects to the global Apex Router automatically.

The Prover listens on TCP 8003, receives relayed mobile witness payloads, compresses proofs, and submits settlement to the L0 Sequencer.

Desktop Prover — Compute Settlement Flow

01 Listen — Prover opens TCP 8003 and awaits mobile witness payloads relayed by the L0 Router via POST /api/relay/payload.
02 Prove — Receives sensor data package. Runs RISC Zero STARK compression. Generates a cryptographic output hash.
03 Upload — Proof artifact uploaded to distributed storage. Settlement payload assembled locally.
04 Settle — POST /api/compute/settle submitted to L0 Sequencer with proof hash and your payout address. L0 records ZK-STARK VERIFIED and credits USDC reward.
Path B · Advanced

entropy-k-core Daemon — Build Guide

For server operators and headless Linux deployments. Requires Rust 1.75+ and the RISC Zero toolchain. Recommended for operators targeting high throughput or custom infrastructure.

System Requirements

  • OSLinux (Ubuntu 22.04/24.04 recommended) or WSL2
  • CPUModern multi-core x86_64 processor
  • RAM8 GB minimum 16 GB+ recommended
  • DependenciesRust 1.75+, standard build tools, rzup
  • NetworkStable broadband with outbound TCP access on ports 8001 and 8003

Step 01 — Toolchain & Build

Install the RISC-V dependencies and compile the bare-metal host binary.

curl -L https://risczero.com/install | bash
rzup
git clone https://github.com/ezumba/entropy-k-core.git
cd entropy-k-core
export RISC0_BUILD_USE_DOCKER=0
cargo build --release

Step 02 — Identity

Provide a valid Base L2 EVM address to receive automated USDC payouts on Base Mainnet.

Format: 0x...

Use any standard EVM hot wallet. This is your permanent yield destination. Protect the private key locally.

Step 03 — Ignite

Connect to the global Apex Router and inject your payout address.

cargo run --release -- \
  --apex 18.209.174.113:8001 \
  --payout <YOUR_BASE_EVM_ADDRESS>

Your node joins the mesh and begins condensing proofs for the M2M economy.

Step 04 — Verify

[MEMBRANE] Proof Condensed & Radiated

Your node is live and earning. The Sovereign Siphon settles proofs and routes USDC to your wallet. Check the Explorer for your verified status.

Cryptographic Architecture

Your Node Identity

ExergyNet nodes authenticate to the Apex Router using a local keypair generated on first run. Your EVM payout address on Base Mainnet receives USDC yields. Your node public key is embedded in every proof you submit — it is your unforgeable identity on the mesh.

Keypair Genesis

Generated automatically on first run and stored locally. Never transmitted. Your private key never leaves your machine.

USDC Yield Destination

Every proof includes your EVM payout address. USDC is routed directly to this address by the L0 Sequencer on Base Mainnet (eip155:8453).

SOL Yield (Legacy Path)

ElizaOS-path operators retain their Solana Ed25519 keypair as the native yield destination. SOL is routed directly with no bridging or wrapping required.
