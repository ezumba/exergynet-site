Development Progress

Roadmap

ExergyNet is built in phases. A phase exits only when its public verification conditions are met.

Phase 7 · Completed

Mainnet settlement foundation

  • Deploy LNES-01 program reference to Solana Mainnet-Beta
  • Execute and publish confirmed settlement proof transaction
  • Publish SDK package: lnes-agent-sdk-core
  • Publish core documentation and verification pages
Phase 8 · Completed

Public discovery and machine-readable metadata

  • Launch public website at exergynet.org
  • Publish /.well-known/exergynet.json
  • Publish /llms.txt for LLM and agent discovery
  • Publish token metadata endpoint /exg-metadata.json
  • Add explorer, docs, SDK, MCP, proof, and security pages
Phase 9 · Active

L0 explorer and producer telemetry

  • Run live ExergyNet L0 explorer at /explorer.html
  • Expose HTTPS explorer API through explorer-api.exergynet.org
  • Track block height, producer identity, thermal anchors, reward, hash, and ledger state
  • Stabilize persistent node identity across restarts
  • Support multi-producer block ingestion and producer-level search
Phase 10 · Active R&D

LNES-03 recursive aggregation

  • Validate local recursive proof prototype
  • Aggregate child proof outputs into a parent proof path
  • Map recursive proof state to future batch settlement logic
  • Design batch settlement interfaces for multi-agent useful compute
Phase 11 · Planned

External agent integrations

  • Package MCP tools for autonomous agent runtimes
  • Add examples for agent frameworks and local LLM harnesses
  • Publish reference integration scripts
  • Document operational safety, keys, limits, and expected-value gating
Phase 12 · Planned

Security hardening and audit readiness

  • Replace text node identities with cryptographic signing keys
  • Verify block signatures before crediting rewards
  • Separate test telemetry from production settlement state
  • Prepare third-party audit package and reproducibility notes
