Step 0 — Critical prerequisite

Confirm your shell environment

All commands in this guide are Bash syntax. Running them in Windows PowerShell will produce parse errors. Verify your environment before proceeding.

✓ Correct: WSL Bash

Your prompt looks like:

username@DESKTOP:~$

Launch from PowerShell by typing wsl and pressing Enter.

✗ Wrong: PowerShell

Your prompt looks like:

PS C:\Users\username>

PowerShell does not support &&, sh, or source.

⚠ If you see The token '&&' is not a valid statement separator or sh: not recognized, you are in PowerShell. Type wsl to switch before continuing.
Installation sequence

Sequential setup steps

Execute each block in order. Do not skip. Each step validates the next.

01
Install build dependencies
GCC toolchain, curl, and Python3 are required for Rust and RISC Zero compilation.
ℹ If you see Could not get lock /var/lib/dpkg/lock-frontend, another process is updating packages. Wait 2 minutes and retry. Do not delete the lock file.
bash 
sudo apt-get update && sudo apt-get install -y build-essential curl python3

Expected: Package lists updated. No errors. sudo will prompt your WSL user password.

02
Install Rust compiler
Installs rustup, Cargo, and the stable x86_64 Linux toolchain.
bash 
curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh -s -- -y

Expected last line: stable-x86_64-unknown-linux-gnu installed - rustc 1.95.0

03
Activate Rust in current session
Adds Cargo's bin directory to PATH. Required every new terminal unless you restart.
bash 
source $HOME/.cargo/env
verification 
rustc --version
# Expected: rustc 1.95.0 (59807616e 2026-04-14)
⚠ PowerShell equivalent does not exist. The command $env:PATH = [System.Environment]... is PowerShell-only syntax and will cause a bash parse error. Use source in WSL only.
04
Install RISC Zero ZK engine
Installs rzup, cargo-risczero, r0vm, and the RISC Zero Rust toolchain v1.94.1.
bash · run as one block 
curl -L https://risczero.com/install | bash
~/.risc0/bin/rzup install
export PATH="$HOME/.risc0/bin:$PATH"
echo 'export PATH="$HOME/.risc0/bin:$PATH"' >> ~/.bashrc

Expected: ✓ Installation complete! rzup is installed and four components installed: cargo-risczero, cpp, r0vm, rust.

05
Forge node architecture
Python script writes the complete Rust workspace to ~/entropy-k-core. Atomic single operation.
bash · paste entire block 
mkdir -p ~/entropy-k-core && cd ~/entropy-k-core

Then paste the full Python scaffold script from the deployment manifest. Expected output:

expected output
>>> ATOMIC WRITE COMPLETE. NODE READY FOR IGNITION.
06
Compile and ignite
Full release build. Downloads 379 crates (~50MB). Guest circuit compiles separately for RISC-V target. Allow 20 minutes.
bash 
cd ~/entropy-k-core
cargo run --release -- 8003 18.209.174.113:8001
✓ Successful ignition output begins with [EXERGYNET] PROOF-OF-EXERGY NODE IGNITED followed by your wallet address and block forge loop.
Diagnostic reference

Known error index

All errors encountered during live deployment. Match your error message to the left column.

Error Signal Root Cause Resolution
token '&&' is not a valid statement separator Command run in PowerShell instead of WSL Bash Type wsl in PowerShell to enter Bash. Do not run Bash syntax in PowerShell.
sh: not recognized as the name of a cmdlet PowerShell has no sh binary Same as above. Switch to WSL environment first.
syntax error near unexpected token '(' PowerShell-only syntax pasted into Bash Remove $env:PATH = [System.Environment]::... lines. Use source $HOME/.cargo/env instead.
Could not get lock /var/lib/dpkg/lock-frontend Ubuntu's unattended-upgrades is running (PID 485) Wait 2 minutes and re-run sudo apt-get install. Never delete the lock file.
3 incorrect password attempts WSL user password entered incorrectly for sudo Re-run command. Enter the password you set when configuring WSL Ubuntu. It is not your Windows password.
winget not found winget is a Windows tool; does not exist in WSL Use apt for package management inside WSL. winget is irrelevant here.
warn: existing rustup settings file at ~/.rustup/settings.toml Rust was previously installed on this WSL instance Non-fatal. Rustup will use the existing toolchain. Proceed normally.
! Version 3.0.5 of r0vm is already installed RISC Zero was previously installed Non-fatal. rzup sets 3.0.5 as default. No action needed.
warning: unused import: Duration Unused import in host/src/main.rs (line 12) Non-fatal compiler warning only. Does not block compilation. Run cargo fix --bin "host" to suppress.

  • [✓] rustc --version returns 1.95.0 or higher
  • [✓] ~/.risc0/bin/rzup exists and is on PATH
  • [✓] ~/entropy-k-core/Cargo.toml exists
  • [✓] cargo run --release completes without error
  • [✓] Node outputs [EXERGYNET] PROOF-OF-EXERGY NODE IGNITED
  • [✓] ledger_state.json shows incrementing balance
